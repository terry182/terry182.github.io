Tarjan's Strongly Connected Components Algorithm

這是一個圖論裏的演算法，用處是求出有向圖或是無向圖的強連通分量。 要理解這個演算法之前，先要知道什麼是強連通分量

強連通分量

對於一個圖\(G=(V, E)\), 若\(\forall u, v \in V\)都存在由\(u\)到\(v\)的路徑和由\(v\)到\(u\)的路徑，則\(G\)是一強連通圖。 而對每一個極大強連通子圖\(S \subset G\)，稱S是G的強連通分量。

這個演算法的核心思想在於每一個強連通分量都是由一個或是多個環所構成。
因此在強連通分量中的某一個結點開始，進行深度優先搜索。
若是按照遍的順序來給每一個結點一個timestamp的話，這樣有一結點能到達的結點的Timestamp可能比自已的還要小。

若是以\(low(i)\)來表示結點\(i\)能到達的timestamp最小的結點，則對屬於同一強連通分量的結點\(u, v\), 會有\(low(u) = low(v)\).，這樣就能分開不同的強連通分量了。

而且每一個通連通分量都是DFS搜索樹的一棵子樹, 因此可以在回溯的時候找出同一個強連通分量中的所有結點，而且可以知道這個子樹是以\(low(u)\)為根。

這是演算法的Pseudo Code:

procedure tarjan(node u)
    index <- index + 1
    timestamp[u] <- index
    low[u] <- index
    Stack.push(u)
    inStack[u] <- True

    for each (u, v) in E do
        if v is not visited then
            tarjan(v)
            low[u] <- min(low[u], low[v])
        else if inStack[v] then  # v is in Stack
            low[u] <- min(low[u], timestamp[v])

    if low[u] = timestamp[u] then
        repeat
            w <- Stack.pop()
            w.inStack = False
            add w in the same strongly connected component
        until w = u

因為DFS可以一個stack實現, 因此可以Stack來處理回溯時在同一個強連通分量的所有結點（以u為根的子樹）。
